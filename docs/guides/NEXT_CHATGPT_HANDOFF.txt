Critical DTED debugging history
-------------------------------
Earlier versions were wrong because:
- they used the wrong point-grid right edge / lower edge values
- or skipped the needed point-grid correction
- which caused the corner-coordinate warning

The corrected logic used in finalbuild is:
- warp to the 1x1 degree cell
- compute half-pixel expansion from width and height
- apply correct point-grid bounds
- translate to DTED

Correct point-grid bounds formula
---------------------------------
For a 1x1 degree tile:
- xres = 1 / (width - 1)
- yres = 1 / (height - 1)
- ulx = west - xres/2
- uly = north + yres/2
- lrx = east + xres/2
- lry = south - yres/2

If future DTED warnings reappear
--------------------------------
Check whether the current script still uses:
- `lrx = east + half_x`
not
- `lrx = east - half_x`

If warning lines appear again, compare against the validated W155/W156 logic first before trying other fixes.

Validation checks for DTED
--------------------------
After a run, verify:
```bash
gdalinfo /path/to/w156/n49.dt2 | grep "Size is"
gdalinfo /path/to/w156/n50.dt2 | grep "Size is"
```

Expected:
- n49 -> Size is 3601, 3601
- n50 -> Size is 1801, 3601

Also verify the latest log has no:
`corner coordinates not properly aligned`
